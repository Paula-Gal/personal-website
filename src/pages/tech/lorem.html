Why fetch() Doesn't Work on file://

If you’ve ever tried to load a local file using fetch(), you probably ran into a cryptic error or found nothing happening at all. This isn’t a bug — browsers intentionally block fetch requests to file:// URLs.

Why it happens

Browsers isolate local files for security reasons. Allowing JavaScript to freely read files on your disk would be a huge vulnerability.

How to fix it

  • Run a small local server (Live Server, Vite, Python HTTP server)
  • Use paths like /assets/data.json instead of absolute file URLs

Once you run your project through a local server, fetch() will behave exactly as expected.
